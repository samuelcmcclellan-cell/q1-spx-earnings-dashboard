# Q1 2026 S&P 500 Earnings — FactSet Dashboard

Two artifacts from one source PDF:

1. **Excel workbook** (`output/Q1_2026_Earnings_Dashboard.xlsx`) — 9-sheet workbook for offline analysis.
2. **Static website** (`web/`) — deployed to Vercel; mirrors the workbook and offers it as a download.

Both are regenerated each week by parsing FactSet's `EarningsInsight_<MMDDYY>.pdf` directly. Every figure is verbatim from the source — no editorial framing, no curated outliers, all 11 GICS sectors shown wherever FactSet reports them.

## Weekly workflow

```bash
# 1. Save the new FactSet PDF to ~/Downloads/EarningsInsight_<MMDDYY>.pdf
# 2. Regenerate the workbook + parsed JSON:
node src/update.mjs "C:/Users/samue/Downloads/EarningsInsight_050126.pdf"

# 3. Commit and push — Vercel will redeploy automatically:
git add data/parsed-*.json output/Q1_2026_Earnings_Dashboard.xlsx
git commit -m "Refresh: 2026-05-01"
git push
```

The site reads from the most recent `data/parsed-MMDDYY.json` automatically (see `web/prebuild.mjs`).

## Repository layout

```
src/
  parse-factset.mjs     PDF text → structured JSON (regex-driven, page-aware)
  build-dashboard.mjs   JSON → 9-sheet .xlsx (single neutral palette)
  update.mjs            CLI entry: node src/update.mjs <pdf-path>
  schema.md             Documents the parser/builder contract

data/
  parsed-MMDDYY.json    One per weekly snapshot — kept for audit/diff

output/
  Q1_2026_Earnings_Dashboard.xlsx   Regenerated each run

web/
  app/                  Next.js App Router pages (Overview, Sectors, Companies, Forward, Valuation, Raw Data)
  components/           StatCard, Bar, Header, SectionHeader
  lib/data.js           Generated by prebuild.mjs from latest parsed JSON
  prebuild.mjs          Runs before next dev/build to sync data + xlsx
  next.config.mjs       Static export config (output: 'export')
```

## Objectivity rules (enforced in code)

1. **No narrative section titles** — sheet/page names match FactSet's own terminology.
2. **No curated outlier framing** — all 11 sectors appear in every sector table.
3. **Single neutral palette** — blue data bars only. No red/green good/bad coloring.
4. **Historical context always shown** — 5-yr / 10-yr averages alongside every current value FactSet provides them for.
5. **Every panel cites source** — PDF filename, page number, FactSet as-of date, parsed-on date.
6. **No interpretive prose** — cell labels match FactSet's own labels.

## Local development

```bash
# Install both root (CLI) and web deps:
npm install
cd web && npm install

# Run the site locally:
cd web && npm run dev
# → http://localhost:3000
```

## Deployment

The site is a Next.js static export (`output: 'export'`) — Vercel hosts it as plain files, no server runtime required.

- **Vercel Root Directory:** `web`
- **Build Command:** `npm run build` (auto-detected)
- **Output Directory:** `out` (auto-detected from `output: 'export'`)
- Pushing to `main` triggers a redeploy.
